Introduction

Base URL: http://localhost

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

This API is not authenticated.

Employee management

APIs for managing employees

Get all employees

requires authentication

This endpoint lets you get all employee data.

Example request:
curl --request GET \
    --get "http://localhost/employees" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json"
const url = new URL(
    "http://localhost/employees"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

fetch(url, {
    method: "GET",
    headers,
}).then(response => response.json());

Create a employee

requires authentication

This endpoint lets you create a employee.

Example request:
curl --request POST \
    "http://localhost/employees" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"full_name\": null,
    \"job\": \"1\",
    \"hire_date\": \"15-11-2023\",
    \"gender\": \"P\",
    \"date_of_birth\": \"15-11-2023\",
    \"email\": \"otto74@example.com\",
    \"phone\": \"0821xxxxxxxx\",
    \"salary\": 8000000
}"

const url = new URL(
    "http://localhost/employees"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "full_name": null,
    "job": "1",
    "hire_date": "15-11-2023",
    "gender": "P",
    "date_of_birth": "15-11-2023",
    "email": "otto74@example.com",
    "phone": "0821xxxxxxxx",
    "salary": 8000000
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Request      

POST employees

Headers

Content-Type      

Example: application/json

Accept      

Example: application/json

Body Parameters

full_name   string   

The full name of the employee.

job   string  optional  

The id of the job. Example: 1

hire_date   string   

Must be a valid date in the format d-m-Y. Example: 15-11-2023

gender   string   

Example: P

Must be one of:
  • L
  • P
date_of_birth   string   

Must be a valid date in the format d-m-Y. Example: 15-11-2023

email   string  optional  

Must be a valid email address. Example: otto74@example.com

phone   string  optional  

The phone number of the employee. Example: 0821xxxxxxxx

address   string  optional  
salary   number  optional  

The salary of the employee. Example: 8000000
